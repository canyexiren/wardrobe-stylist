# 需求规格说明书

> 版本: v1.0 | 日期: 2026-06-07 | 状态: 开发中

## 1. 项目背景

### 1.1 项目名称
衣橱穿搭助手 (Wardrobe Stylist)

### 1.2 项目定位
一款运行在浏览器中的个人衣橱管理 + 智能穿搭推荐工具。帮助用户管理自己的衣物，根据实时天气和温度自动推荐合适的穿搭组合。

### 1.3 目标用户
- 日常需要搭配衣服的普通用户
- 想根据天气科学穿衣的用户
- 希望系统管理衣橱的用户

---

## 2. 功能需求

### 2.1 底部导航（全局）

系统包含三个主页面，通过底部导航栏切换：

| Tab | 图标语义 | 名称 | 功能概括 |
|-----|---------|------|---------|
| 1 | ✨/灯泡 | 穿搭灵感 | 查看天气、选择风格、生成穿搭建议 |
| 2 | 👗/衣柜 | 我的衣橱 | 衣物照片管理（增删改查） |
| 3 | 👤/设置 | 个人中心 | 偏好设置、数据备份恢复 |

### 2.2 穿搭灵感页

#### 2.2.1 天气展示区
- **自动获取**: 通过浏览器定位获取用户位置，调用天气API获取实时数据
- **展示内容**: 天气图标（晴/多云/雨/雪等）、当前温度、天气描述
- **降级方案**: 当无法获取位置时，提供手动输入温度入口

#### 2.2.2 风格选择器
- 三个选项单选：**通勤** / **休闲** / **精致**
- 默认选中用户在个人中心设置的偏好风格
- 切换时应有视觉反馈

#### 2.2.3 穿搭生成
- **一键生成按钮**: 点击后根据天气+温度+风格自动生成穿搭
- **穿搭结果**: 按分类（上衣+外套、下装、鞋子等）展示推荐单品卡片
- **换一换按钮**: 对某分类结果不满意时，换一个同分类的其他匹配单品
- **空衣橱提示**: 衣橱为空时引导用户先去添加衣物

### 2.3 我的衣橱页

#### 2.3.1 分类筛选
- 筛选标签：全部 / 上衣+外套 / 下装 / 连衣裙+连体衣 / 鞋子+配饰
- 选中标签高亮
- 支持查看各分类下的衣物数量

#### 2.3.2 衣物展示
- 网格布局展示衣物卡片（照片在上、名称在下）
- 空衣橱时显示引导提示

#### 2.3.3 添加衣物
- 入口：页面右下角浮动"+"按钮
- 表单字段：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| 照片 | 图片上传 | 否 | 支持本地文件选择，显示预览 |
| 名称 | 文本输入 | 是 | 衣物名称，方便识别 |
| 分类 | 下拉选择 | 是 | 上衣+外套 / 下装 / 连衣裙+连体衣 / 鞋子+配饰 |
| 颜色 | 文本/颜色选择 | 否 | 主要颜色描述 |
| 保暖度 | 单选 | 是 | 薄款 / 适中 / 厚款 / 加厚 |
| 风格标签 | 多选 | 是 | 通勤 / 休闲 / 精致 |
| 适合季节 | 多选 | 否 | 春 / 夏 / 秋 / 冬 |

#### 2.3.4 编辑衣物
- 点击衣物卡片 → 弹出编辑表单（预填已有数据）
- 可修改所有字段
- 保存更新

#### 2.3.5 删除衣物
- 编辑表单中提供"删除"按钮
- 点击后弹出确认弹窗
- 确认后删除并刷新列表

### 2.4 个人中心页

#### 2.4.1 默认风格偏好
- 设置默认的风格选择（通勤/休闲/精致）
- 保存后在穿搭灵感页默认选中

#### 2.4.2 温度单位
- 摄氏度 / 华氏度切换
- 切换后全局温度显示同步更新

#### 2.4.3 数据导出
- 一键导出所有衣物数据为 JSON 文件
- 下载到本地作为备份

#### 2.4.4 数据导入
- 选择之前导出的 JSON 文件
- 恢复衣物数据
- 导入前提示将覆盖现有数据

#### 2.4.5 使用说明
- 简单的文字说明，帮助用户了解如何使用

## 3. 非功能需求

### 3.1 性能
- 页面加载时间 < 2 秒
- 图片压缩处理，单张不超过 500KB（存储为 base64）

### 3.2 兼容性
- 支持 Chrome 90+
- 支持 Edge 90+
- 支持移动端浏览器（响应式）

### 3.3 数据安全
- 所有数据存储在用户浏览器本地（IndexedDB）
- 不上传到任何服务器
- 用户可随时导出备份

### 3.4 可用性
- 界面简洁直观，无需学习成本
- 所有操作有即时反馈
- 错误情况有友好提示

## 4. 验收标准

- [ ] 三个Tab页面可正常切换
- [ ] 天气自动获取并正确显示
- [ ] 衣物可添加、编辑、删除
- [ ] 衣物按分类筛选正确
- [ ] 穿搭生成结果合理（匹配温度和风格）
- [ ] 数据刷新后不丢失
- [ ] 数据可导出和导入恢复
- [ ] 页面在手机和桌面浏览器均正常显示

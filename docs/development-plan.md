# 分阶段执行计划

> 版本: v1.0 | 日期: 2026-06-07

## 执行原则

1. **小步推进**: 每次只做一个小阶段，完成并验证后再进入下一阶段
2. **先跑通再优化**: 优先实现核心功能，UI打磨放在最后
3. **每阶段验证**: 完成阶段产出后立即在浏览器中验证
4. **日志记录**: 每次开发后更新 `dev-logs/` 中的日志文件
5. **变更记录**: 需求或设计变更时更新对应的 `docs/` 规范文档
